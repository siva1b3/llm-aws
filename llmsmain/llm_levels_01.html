LLM Inference — 50-level Incremental Build

Target: one inference, one token predicted. No KV cache, no batching. Each level adds exactly one new concept. Highlighted rows mark what changed vs. the previous level.
Columns: Step · Runtime in · Static data · Runtime out. A Shape column appears at Level 49.

Notation & Sizes
N_tok = number of input tokens (variable per inference) d = per-token vector dim (e.g., 4096) d_ff = FFN hidden dim (e.g., 14336) n_h = number of query heads (e.g., 32) n_kv = number of K/V heads (GQA) (e.g., 8) d_h = per-head dim = d / n_h (e.g., 128) vocab_size = vocabulary size (e.g., 128256) N = number of transformer blocks (e.g., 32) max_pos = max supported positions (e.g., 8192)

Phase 1 — Outer pipeline (around the core)

Level 0 — opaque box

New: Entire pipeline as one box. {model} bundles everything — vocab plus all learned weights.
